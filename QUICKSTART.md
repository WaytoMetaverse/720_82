## 🎮 如何使用

### 1. 查看360度环景
- 👆 **点击并拖动**：左右拖动旋转查看，支持无限循环
- 📱 **触摸滑动**：在移动设备上同样支持
- 🖥️ **全屏体验**：无边框全屏显示，沉浸式观看

### 2. 切换空间
点击顶部浮动导航栏的按钮：
- 🏠 **客餐厅**：可以切换沙发和茶几
- 🛏️ **主卧室**：查看卧室环境
- 🛏️ **次卧室**：查看次卧环境

### 3. 切换家具（客餐厅）

系统会自动识别ID图中的色块区域：

#### 🟢 绿色区域 = 沙发
- 鼠标悬停会显示黄色高光
- 点击切换沙发款式（A → B → C → A...）

#### 🔴 红色区域 = 茶几
- 鼠标悬停会显示黄色高光
- 点击切换茶几款式（A → B → C → A...）

### 4. 查看当前状态
底部的控制面板会显示：
- 🎨 色块标识
- 📝 物件名称
- 🔢 当前款式和总数

## 🔍 ID图色块说明

你的ID图使用以下颜色标识可点击区域：

| 颜色 | 物件 | RGB值 | 用途 |
|------|------|-------|------|
| 🟢 绿色 | 沙发 | (0, 255, 0) | 点击切换沙发款式 |
| 🔴 红色 | 茶几 | (255, 0, 0) | 点击切换茶几款式 |
| ⬛ 黑色 | 背景 | (0, 0, 0) | 不可点击区域 |

⚠️ **重要**：ID图必须使用精确的RGB值，不能有压缩！

## 🐛 常见问题

### Q: 点击没有反应？
**A:** 检查以下几点：
1. 确认ID图文件存在且命名正确
2. ID图必须是PNG格式（你的是JPG，可能需要转换）
3. RGB值必须精确匹配：绿色(0,255,0)、红色(255,0,0)

### Q: 高光位置不准确？
**A:** 
- 确认全景图和ID图尺寸完全一致
- 检查ID图是否被压缩过

### Q: 图片加载失败？
**A:** 
- 检查浏览器控制台的错误信息
- 确认文件路径正确：`panorama/客餐廳_A沙發_A茶几.jpg`
- 确认文件编码正确（中文文件名）

### Q: 主卧室和次卧室没有可点击物件？
**A:** 这是正常的！目前这两个空间配置为只有环境浏览，没有可切换的家具。如果你的ID图中有标记物件，可以在 `script.js` 中添加配置。

## 📝 添加更多物件

如果你的主卧室或次卧室ID图中也有色块标记，可以这样配置：

打开 `script.js`，找到配置部分，修改：

```javascript
master: {
    name: '主卧室',
    color: '#FFFF00',
    panorama: null,
    idMap: null,
    objects: [
        // 添加你的物件
        { 
            id: 'bed', 
            name: '床', 
            color: '#00FF00', 
            rgb: [0, 255, 0], 
            variants: ['A', 'B'], 
            currentVariant: 0 
        }
    ]
}
```

## 🚀 部署到GitHub Pages

### 步骤1：创建GitHub仓库

```bash
git init
git add .
git commit -m "初始提交：环景导览系统"
git branch -M main
git remote add origin https://github.com/你的用户名/仓库名.git
```

1. 进入仓库的 **Settings**
2. 左侧菜单找到 **Pages**
3. Source选择：**main** 分支，**/ (root)** 文件夹
4. 点击 **Save**
5. 等待1-2分钟

```
https://你的用户名.github.io/仓库名/
```
